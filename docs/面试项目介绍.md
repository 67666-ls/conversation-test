# 美团AI Hackathon 项目面试介绍

---

## 30秒版（开场或简历过时用）

> 我参加了美团AI Hackathon，做的是命题二——AI外呼评测系统。核心思路是用两个大模型互相对话：一个扮演外呼客服Agent，一个扮演用户，模拟真实通话场景。系统能自动从任务完成度、沟通质量、合规性等5个维度打分，还有人工审核队列做兜底。整个系统用FastAPI做后端，前端是Alpine.js的单页应用，接的是DeepSeek API。

---

## 深聊版

### 一、项目背景（Why）

传统外呼质检靠人工抽检，一天几千通电话只能抽几十条，效率很低。我们想用AI解决这个问题——不只是录音转文字，而是真正能理解"这通电话打得好不好"。

### 二、核心设计（What/How）

**1. 双Agent架构**

最有意思的地方是我们用了双Agent对打。主Agent按照任务指令扮演外呼客服，用户模拟器Agent扮演客户，两个模型对话直到通话结束。用户模拟器支持12种人设——配合型、抗拒型、急躁型、老年型、愤怒型等，覆盖真实用户的典型行为。

**2. 评测引擎（5维评分）**

| 维度 | 权重 | 评分依据 |
|------|------|----------|
| 任务完成度 | 35% | 指令关键词覆盖 + 信息收集完整性 |
| 沟通质量 | 25% | 话术流畅度 + 信息传达准确度 |
| 合规性 | 20% | 违禁词命中 + 合规话术使用 |
| 效率 | 10% | 通话轮次 + 有效信息密度 |
| 用户体验 | 10% | 语气友好度 + 合理诉求响应 |

每个维度不仅给分，还输出详细依据（如"关键词命中4/5，缺失：优惠力度"），方便后续优化话术。

**3. 人工审核兜底**

完全靠规则评分会有误判，所以设计了人工审核队列。低分或触发特定词的会话自动标记，审核员可以通过/驳回/手动改分，相当于机器初筛+人工复核的双保险流程。

### 三、技术选型

| 层 | 技术 | 选型理由 |
|----|------|----------|
| 后端 | FastAPI | 异步高性能，自动生成API文档 |
| 数据库 | SQLite | 比赛场景轻量够用，可无缝切换PostgreSQL |
| 前端 | Alpine.js SPA | 单文件HTML无需打包，开发效率高 |
| AI | DeepSeek API | 成本低，对话质量够用 |
| 加密 | bcrypt | 直接调用，绕过passlib兼容问题 |

### 四、遇到的挑战（加分项）

**1. passlib与bcrypt版本冲突**
注册登录一直报错，排查后发现passlib 1.7的`__about__`属性在新版bcrypt里不存在了，最后绕过passlib直接用bcrypt原生API解决。

**2. Alpine.js状态管理**
单文件SPA没有路由，弹窗关闭时tab状态会漂移，用`setTimeout`替代`$nextTick`来保证DOM更新顺序。

**3. Windows端口预留**
8001端口被系统预留，排查发现是Win NAT排除范围冲突，改为8088解决。

### 五、成果数据

使用12种人设 × 2个场景生成了24条模拟对话做验证：

- 平均综合得分：**73.2分**
- 等级分布：B级19条 / C级5条
- 触发审核率：37.5%
- 信用卡场景优于健康险（关键词易覆盖 vs 合规要求严）
- 配合型得分最高，抗拒型/愤怒型最低

## 可能被追问的问题

### Q1: 为什么用双Agent而不是直接评录音？

可以主动构造各种极端场景，不依赖真实数据，成本低且可控。而且我们能精确控制输入变量，做A/B测试对比不同话术效果。

### Q2: 合规性怎么判断的？

目前维护违禁词库，命中即扣分。规则扩展性有限，后续可以接LLM做语义判断。这块是已知的待优化点。

### Q3: DeepSeek和GPT-4怎么选的？

成本和速度。DeepSeek每百万token便宜很多，对话质量在这个场景下够用。而且国产模型在中文语境下表现反而更自然。

### Q4: 有没有考虑过量产？

有想过。接真实录音的话，前端加个ASR转文字模块，后端评测引擎不需要大改。数据库从SQLite换PostgreSQL，加个任务队列就行。

### Q5: 最大的不足是什么？

评测维度权重是经验设定的，没有数据驱动的校准。另外规则引擎vs语义引擎的trade-off还没解决——纯规则效率高但不够准，纯LLM准确但成本太高。

### Q6: 如果重新做一次会怎么改进？

第一，引入语义相似度替代纯关键词匹配；第二，用实际质检数据校准各维度权重；第三，加个A/B实验框架，能对比不同话术版本的效果。
